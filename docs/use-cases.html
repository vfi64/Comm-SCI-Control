Application matrix

Choose the mode for the task. The framework improves fit, not quality in a universal way.

The strongest levers are usually profile choice, SCI on/off, and Strict/Explore overlay. Use the smallest combination that still fits the question type.

Quick defaults

Five robust starting points

Explain

`Profile Standard` + `Strict on`

Deep check

`Profile Expert` + `SCI on` (A) + `Strict on`

Challenge

`Profile Sparring` + `Strict on`

Explore

`Profile Sandbox` + `Explore on`

Condense

`Profile Briefing` + `Strict on`

Normal mode

`Comm Stop`

Question patterns

What to pick and why

Conceptual depth

`Expert + SCI A/B + Strict` for high-precision concept work (for example: "What is time?").

Controversial claims

`Sparring + Strict` or `Expert + SCI E/G + Strict` for counter-arguments and risk checks.

Ideation

`Sandbox + Explore` for search space and unconventional options.

Didactic simplification

`Standard + Strict` or `Briefing + Strict` for concise, teachable outputs.

Practical notes

Keep complexity proportional

SCI recurse is a specialist tool for genuinely nested multi-part tasks. Color affects scanability, not epistemic quality. For most workflows, `Standard + Strict` is the safest default.
